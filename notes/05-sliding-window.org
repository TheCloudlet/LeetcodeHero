* 1. Primitives
The basic elements of Sliding Window are not data structures, but conceptual atomic operations:

- Window Boundaries (left, right) :: Define the range of the current window, usually represented by indices.

- Window State :: The aggregated representation of data within the window (e.g., character frequency map, sum, set).

- Expand (right++) :: Expand the window to the right by one element.

- Shrink (left++) :: Shrink the window from the left by one element.

- Valid / Invalid Condition :: Determine whether the current window meets the problem requirements (e.g., all characters are unique, sum is less than the target).

These primitives can be demonstrated using pseudocode or C++ code:

#+BEGIN_SRC cpp
int left = 0, ans = 0; // 1. Window Boundaries
State window;          // 2. Window State (e.g., unordered_map, sum)

for (int right = 0; right < n; ++right) {
  // 3. Expand: Update window state with s[right]
  window.add(s[right]);

  // 5. Valid / Invalid Condition
  while (!window.is_valid()) {
    // 4. Shrink: Remove s[left] from state and increment left
    window.remove(s[left]);
    left++;
  }

  // Window [left, right] is now valid, process or record answer
  ans = max(ans, right - left + 1);
}
#+END_SRC

** 2.1 Fixed-Length Window
The window size is fixed to k, typically used for calculating moving averages, maximum values, etc.

We can use the template to maintain the valid condition. But I think using accumulate is a better option.

#+NAME: fixed-length-window
#+BEGIN_SRC cpp
// LC643. Maximum Average Subarray I
double sum = std::accumulate(nums.begin(), nums.begin() + k, 0.0);
double max_sum = sum;

for (int pos = k; pos < n; ++pos) {
  sum = sum + nums[pos] - nums[pos - k];
  max_sum = std::max(max_sum, sum);
}
#+END_SRC

** 2.2 Variable-Length Window (Finding Longest / Shortest)
Longest substring: Expand until the condition is violated, then shrink until the condition is restored, recording the window length.
Shortest substring: Expand until the condition is met, then shrink until the condition is just broken, recording the length.
Classic problems: LC 3 (Longest Substring Without Repeating Characters), LC 209 (Minimum Size Subarray Sum).
#+NAME:
#+BEGIN_SRC cpp
int left = 0;
for (int right = 0; right < n; ++right) {
  // add s[right] to window state
  while (/* condition violated */) {
    // remove s[left] from window state
    left++;
  }
  // window is valid, update answer
}
#+END_SRC

** 2.3 Counting Window (e.g., Shortest substring containing all characters)
Use need and have counters to track the fulfillment of target characters.
Classic problems: LC 438 (Find All Anagrams in a String), LC 567 (Permutation in String).
#+NAME:
#+BEGIN_SRC cpp
int need = target.size(), have = 0;
while (right < n) {
  // expand
  if (freq[s[right]]++ == 0) have++;
  right++;

  while (have == need) {
    // shrink to find minimal
    if (freq[s[left]]-- == 1) have--;
    left++;
  }
}
#+END_SRC

** 2.4 Multi-Condition Window (e.g., At most k distinct characters)
Use a frequency map to track the number of distinct characters; shrink when it exceeds k.
Classic problems: LC 340 (Longest Substring with At Most K Distinct Characters), LC 904 (Fruit Into Baskets).

* 2. Means of Combination & Abstraction* 2. Means of Combination & Abstraction
在定義了 Primitives 之後，我們需要將它們組合並抽象化。Sliding Window 本質上是一個維持不變性 (Invariant) 的狀態機 (State Machine)。

** Means of Combination (組合的方法)
我們透過控制流 (Control Flow) 將 Primitives 組合起來。最經典的組合方式是 =for= 迴圈包裹 =while= 迴圈：
- *外層驅動 (The Clock)*：=right= 指標不斷前進，如同硬體的 Clock cycle，持續將新資料注入狀態機 (Expand)。
- *內層反饋 (The Feedback Loop)*：當注入新資料導致狀態違反了 =is_valid()= 條件時，觸發 =while= 迴圈，持續推動 =left= 指標 (Shrink)，直到狀態機再次回到穩定合法 (Valid) 的狀態。

這種組合方式保證了資料流過時，Window 內的狀態永遠會收斂到一個合法的區間。

** Means of Abstraction (抽象的方法)

#+BEGIN_SRC cpp
// Abstraction Barrier: The Window State
class WindowState {
public:
 void add(char c);     // Expand primitive
 void remove(char c);  // Shrink primitive
 bool is_valid();      // Invariant check
};

// The General Process
template <typename Action>
void slidingWindowProcess(const string& s, WindowState& window,
                          Action on_valid_window) {
  int left = 0;
  for (int right = 0; right < s.length(); ++right) {
    window.add(s[right]);

    while (!window.is_valid()) {
      window.remove(s[left]);
      left++;
    }

    on_valid_window(left, right, window);
  }
}

* 5. Common Mistakes
- Forgetting to update the answer after shrinking (only record when the window is valid).
- Confusing the timing of expanding and shrinking.
- Fixed-length window boundary update order: Add first, then check, then remove.
- Using while instead of if for shrinking (might shrink multiple elements at once).
- Not initializing the starting points of left and right correctly.
- Using the wrong condition in counting windows (e.g., using have == need instead of have >= need).
