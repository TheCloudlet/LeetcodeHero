* 1. Primitives
The basic elements of Sliding Window are not data structures, but conceptual atomic operations:

- Window Boundaries (left, right) :: Define the range of the current window, usually represented by indices.

- Window State :: The aggregated representation of data within the window (e.g., character frequency map, sum, set).

- Expand (right++) :: Expand the window to the right by one element.

- Shrink (left++) :: Shrink the window from the left by one element.

- Valid / Invalid Condition :: Determine whether the current window meets the problem requirements (e.g., all characters are unique, sum is less than the target).

These primitives can be demonstrated using pseudocode or C++ code:

#+BEGIN_SRC cpp
int left = 0, ans = 0; // 1. Window Boundaries
State window;          // 2. Window State (e.g., unordered_map, sum)

for (int right = 0; right < n; ++right) {
  // 3. Expand: Update window state with s[right]
  window.add(s[right]);

  // 5. Valid / Invalid Condition
  while (!window.is_valid()) {
    // 4. Shrink: Remove s[left] from state and increment left
    window.remove(s[left]);
    left++;
  }

  // Window [left, right] is now valid, process or record answer
  ans = max(ans, right - left + 1);
}
#+END_SRC

* 2. Means of Combination & Abstraction
#+BEGIN_SRC cpp
// The General Process
template <typename Action>
void slidingWindowProcess(const string& s, WindowState& window,
                          Action on_valid_window) {
  int left = 0;
  for (int right = 0; right < s.length(); ++right) {
    window.add(s[right]);

    while (!window.is_valid()) {
      window.remove(s[left]);
      left++;
    }

    on_valid_window(left, right, window);
  }
}
#+END_SRC

* 5. Common Mistakes
- Forgetting to update the answer after shrinking (only record when the window is valid).
- Confusing the timing of expanding and shrinking.
- Fixed-length window boundary update order: Add first, then check, then remove.
- Using while instead of if for shrinking (might shrink multiple elements at once).
- Not initializing the starting points of left and right correctly.
- Using the wrong condition in counting windows (e.g., using have == need instead of have >= need).
