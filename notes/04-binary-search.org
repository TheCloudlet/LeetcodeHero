** 1.1: Finding the exact number
Finding the index of ~4~ in the sorted array:

#+BEGIN_EXAMPLE
Index:    0    1    2    3    4    5
Data:   [ 1,   2,   3,   4,   5,   6 ]
#+END_EXAMPLE

Steps:
1. Pick a middle.
2. Check whether the number should be in the right or left side of the mid

| # | ~[L, R]~ | ~mid~ | ~val~ | cmp  | Action  |
|---+----------+-------+-------+------+---------|
| 1 | ~[0, 5]~ | ~2~   | ~3~   | < 4  | =L=m+1= |
| 2 | ~[3, 5]~ | ~4~   | ~5~   | > 4  | =R=m-1= |
| 3 | ~[3, 3]~ | ~3~   | ~4~   | == 4 | Found!  |

Code:

#+NAME:
#+BEGIN_SRC cpp
// Leetcode 704 704. Binary Search
#include <vector>

class Solution {
 public:
  int search(const std::vector<int>& nums, int target) {
    int n = static_cast<int>(nums.size());
    int left = 0;
    int right = n - 1;  // Closed-interval
    while (left <= right) {
      const int mid = left + (right - left) / 2;
      if (nums[mid] == target)
        return mid;
      else if (nums[mid] < target)
        left = mid + 1;
      else
        right = mid - 1;
    }

    return -1;  // Cannot find
  }
};
#+END_SRC

** 1.2 Lower Bound: *find the first false*

std::lower_bound :: 在 ~[first, last)~ 半開區間中，尋找第一個「**不符合 (~False~)**」條件的元素。

The concept is we *partition* the number into two halves.
- things we care
- things we don't care

Finding the lower_bound of ~4~ in the sorted array:

#+BEGIN_SRC cpp
#include <algorithm>

// `4` is passed as the `value` into the implicit predicate (element < value).
// std::lower_bound searches for the first iterator where (element < 4) evaluates to False.
auto it = std::lower_bound(nums.begin(), nums.end(), 4);

// The above implicitly does exactly what the following Custom Comparator does:
auto comp = [](int element, int value) {
  return element < value;
};

// Don't forget to pass `comp` as the 4th argument!
auto it2 = std::lower_bound(nums.begin(), nums.end(), 4, comp);
#+END_SRC


Predicate: isSmallerThan4

#+BEGIN_EXAMPLE
Index:         0    1    2    3    4    5
Data:        [ 1,   2,   3,   4,   5,   6 ]
> Predicate:   T    T    T    F    F    F
#+END_EXAMPLE


| # | Search Space ~[L, R)~ | ~mid~ | ~val~ | SmallerThan4P | Action      |
|---+-----------------------+-------+-------+----------------------+-------------|
| 1 | ~[0, 6)~              | ~3~   | ~4~   | ~4 < 4~ (False)      | ~R = mid~   |
| 2 | ~[0, 3)~              | ~1~   | ~2~   | ~2 < 4~ (True)       | ~L = mid+1~ |
| 3 | ~[2, 3)~              | ~2~   | ~3~   | ~3 < 4~ (True)       | ~L = mid+1~ |
| 4 | ~[3, 3)~              | -     | -     | ~L == R~             | Break       |

How to implement our own binary search exact match using lower bound template:
1. Using half-open interval [0, 6)
2. Predicate
3. The ~left = mid + 1~ and ~right = mid~ used ~/2~ (round down) traits and cannot be changed.

#+NAME:
#+BEGIN_SRC cpp
// Leetcode 35. Search Insert Position
#include <vector>

class Solution {
 public:
  int searchInsert(const std::vector<int>& nums, int target) {
    int n = static_cast<int>(nums.size());

    int left = 0;
    int right = n;

    while (left < right) {
      const int mid = left + (right - left) / 2;
      if (nums[mid] < target)  // Predicate
        left = mid + 1;        // DON'T CHANGE SEQUENCE
      else
        right = mid;           // DON'T CHANGE SEQUENCE
    }

    return left;               // right == left, so return anything is okay
  }
};
#+END_SRC

** 1.3 Upper Bound
- For our self defined binary search, by changing predicate (cmp) ~element < value~ (lower_bound) to ~element <= value~ can do the job

- For ~std::upper_bound~ ... They re-implemented it to find the first True

  ~[False, False, False, True, True]~  <- Finds the first True

  #+BEGIN_QUOTE
  *Why does C++ STL ~upper_bound~ feel so weird?*
  
  The C++ Standard Library relies exclusively on *Strict Weak Ordering* (using only ~std::less~ or ~operator<~). It fundamentally avoids demanding ~operator<=~ or ~operator>~ for custom structures.
  
  To find an element strictly greater than ~value~ (i.e., ~element > value~), C++ brilliantly flips the arguments and evaluates ~value < element~!
  
  - ~lower_bound~ evaluates: ~comp(element, value)~ -> Finds the first ~False~ (meaning ~element >= value~, including ~element == value~)
  - ~upper_bound~ evaluates: ~comp(value, element)~ -> Finds the first ~True~ (meaning ~value < element~, strictly excluding ~element == value~ because it equals ~!(element <= value)~)
  
  Array Partition for ~upper_bound~ evaluating ~value < element~:

  [False, False, False, True, True]
                        ^
                  Finds the first True

  By simply flipping the argument order from ~(element, value)~ to ~(value, element)~, C++ elegantly derives the upper bound without ever needing to define a new ~<=~ or ~>~ comparison operator!
  #+END_QUOTE
