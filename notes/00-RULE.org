* 1. ALWAYS UNDERSTAND THE QUESTION
- Identify the Core Question :: Before writing any code, pinpoint the fundamental problem abstraction (e.g., "Is there a loop?" or "How many groups?").
- Verify Input & Output Constraints :: Explicitly clarify the data range, boundary conditions (e.g., empty input, $N=1$), and the required return type.
- Trace a Sample Case :: Manually dry-run the logic with a simple test case to ensure the mental model matches the problem's reality.

* 2. APPLY SYSTEMATIC ABSTRACTION (SICP STYLE)
- Wishful Thinking :: Assume the existence of the sub-procedures you need. Define the high-level logic (e.g., ~expand frontier~) first, and implement the "Primitives" later. This prevents getting bogged down in implementation details while the architecture is still forming.
- Data-flow & Iterative State :: Treat algorithms as a sequence of state transformations ($S_{i} \to S_{i+1}$). Use clear "Frontiers" or "States" to represent the current snapshot of the world, and ensure each iteration is a pure transition from the previous one.
- Avoid Micro-management :: Trust the "Means of Combination" (like the Call Stack or DSU's internal path compression) to handle the complexity. Don't fight the natural flow of the data with unnecessary global state resets.

* 3. NAMING (Google C++ Style)
| Category          | Convention                    | Example                                       |
|-------------------+-------------------------------+-----------------------------------------------|
| Regular Functions | ~PascalCase~                  | ~HasCycle()~, ~FindRedundantConnection()~     |

| Variables         | ~snake_case~                  | ~int course_count~, ~vector<int> indegrees~   |
| Constants         | ~kPascalCase~                 | ~constexpr int kMaxNodes = 1000;~             |
| Member variables  | ~snake_case_~  (trailing ~_~) | ~int size_;~, ~vector<int> adj_;~             |
| Namespaces        | ~lowercase~                   | ~namespace graph_algos~                       |
| Containers        | plural noun                   | ~vector<int> neighbors~                       |
| Counters          | ~_count~ suffix or plural     | ~int edge_count~, ~vector<int> indegrees~     |
| Iterators / temps | short ok in small scope       | ~i~, ~j~, ~it~                                |
